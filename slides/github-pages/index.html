    GitHub Pages 公開手順

    任意のサイトを GitHub Pages で公開するまでの流れを説明します。

    Overview

    GitHub Pages とは

    • GitHub リポジトリから静的サイトを無料で公開できるサービスです。
    • HTML / CSS / JavaScript など、ビルド済みのファイルをそのまま配信できます。
    • 公開 URL は https://<ユーザー名>.github.io/<リポジトリ名>/ です。

    Step 1

    事前準備

    1. GitHub アカウントを作成する
    2. 公開したいサイトのファイルを用意する
    3. GitHub にリポジトリを作成し、ファイルを push する

    トップページは index.html を置いた場所（通常はリポジトリ root）になります。

    Step 2A

    ブランチから公開

    ビルド不要の静的サイト向けの簡単な方法です。

    1. リポジトリの Settings → Pages を開く
    2. Build and deployment で Deploy from a branch を選ぶ
    3. Branch を main、Folder を / (root) に設定
    4. Save を押して数分待つ

    Step 2B

    GitHub Actions で公開

    任意のフォルダ構成や、将来の自動デプロイに向いています。

    1. .github/workflows/ にワークフローを追加する
    2. main ブランチへ push する
    3. Actions の実行が成功すると Pages に反映される

    Settings → Pages の Source は GitHub Actions を選びます。

    Example

    ワークフロー例

    リポジトリ全体をそのまま公開する最小構成です。

    name: Deploy static content to Pages

on:
  push:
    branches: ["main"]

permissions:
  contents: read
  pages: write
  id-token: write

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/configure-pages@v5
      - uses: actions/upload-pages-artifact@v3
        with:
          path: "."
      - uses: actions/deploy-pages@v5

    Step 3

    公開を確認する

    1. Settings → Pages に表示される URL を開く
    2. Actions タブでデプロイが success になっているか確認する
    3. 表示が古い場合は、数分待ってから再読み込みする

    更新が反映されないときは、ブラウザのキャッシュを消して確認してください。

    Tips

    よくあるポイント

    • 公開対象は静的ファイルのみです（PHP や DB は動きません）。
    • リポジトリが Private でも Pages 公開は可能です（プランにより制限あり）。
    • サブディレクトリ配信では、リンクや画像パスを相対パスにすると安全です。
    • 複数デプロイが重なると古い版が表示されることがあります。Actions の最新成功版を確認してください。
